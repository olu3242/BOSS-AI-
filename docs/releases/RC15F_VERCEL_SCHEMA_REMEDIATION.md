## Root Cause

The root `vercel.json` contained an unsupported top-level property:

```json
"projects"
```

The official Vercel `vercel.json` schema sets `additionalProperties: false`; `projects` is not an allowed top-level property.

## Affected File

`C:\Cdev\BOSS AI\BOSS-AI\vercel.json`

Before:

```json
{
  "version": 2,
  "projects": [
    {
      "name": "boss-web",
      "src": "apps/web"
    }
  ]
}
```

After:

```json
{
  "version": 2
}
```

## Exact Change

Removed only the unsupported `projects` property from the root `vercel.json`.

`version` was retained because it is an allowed Vercel schema property. The app-local `apps/web/vercel.json` was not changed because its keys are schema-valid and match the monorepo app layout.
