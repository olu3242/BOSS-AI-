# Capability Operations

Operations begins at release and tracks adoption, installed versions, runtime
health, latency, failures, upgrade success, policy violations, and deprecation
notices. Signals must retain tenant, capability version, correlation ID, and
trace ID without exposing tenant data across boundaries.

Operational evidence informs optimize, upgrade, deprecate, and retire
decisions. Full metric persistence and Decision Ledger integration are planned
BCL implementation work, not part of CPP.
